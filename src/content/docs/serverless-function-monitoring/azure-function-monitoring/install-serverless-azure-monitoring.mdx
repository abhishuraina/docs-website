You can configure Azure Functions monitoring for Linux, Windows, and containerized Azure Functions by instrumenting your Azure Function with the New Relic .NET agent, Python agent, or Node.js agent. 

## Before you begin [#begin]

Ensure that your Azure Function meets our [compatibility and requirements](/docs/serverless-function-monitoring/azure-function-monitoring/compatibility-requirement-azure-monitoring).

<Steps>

<Step>
## Link your Azure account to New Relic [#link]

You can link your Azure account to New Relic by configuring the polling of Azure monitor metrics. This allows you to see the metrics in the New Relic UI. For more information, refer [Azure integration](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-functions-monitoring-integration/#polling).
</Step>

<Step>
## Instrument your Azure Function [#instrument]

Depending on your deployment environment, select one of the following options to instrument your Azure Function with the New Relic.

<Tabs>
  <TabsBar>
    <TabsBarItem id="linux">Linux</TabsBarItem>
    <TabsBarItem id="windows">Windows</TabsBarItem>
    <TabsBarItem id="container">Containerized Function</TabsBarItem>
  </TabsBar>

  <TabsPages>

    <TabsPageItem id="linux">

    You can instrument your Azure Function with the New Relic on Linux in the following ways:.

    * To instrument your `Node.js` runtime with the New Relic, refer to [Node.js agent](/docs/serverless-function-monitoring/azure-function-monitoring/linux/#node).
    * To instrument your `.NET` runtime with the New Relic, refer to [.NET agent](/docs/serverless-function-monitoring/azure-function-monitoring/linux/#net).
    * To instrument your `Python` runtime with the New Relic, refer to [Python agent](/docs/serverless-function-monitoring/azure-function-monitoring/linux/#python).

    
    
    </TabsPageItem>
    <TabsPageItem id="windows">
    You can instrument your Azure Function with the New Relic on Windows in the following ways:.

    * To instrument your `Node.js` runtime with the New Relic, refer to [Node.js agent](/docs/serverless-function-monitoring/azure-function-monitoring/windows/#node).
    * To instrument your `.NET` runtime with the New Relic, refer to [.NET agent](/docs/serverless-function-monitoring/azure-function-monitoring/windows/#net).
    </TabsPageItem>

    <TabsPageItem id="container">
    
    You can instrument your Azure Function with the New Relic on Windows in the following ways:.

    * To instrument your `Node.js` runtime with the New Relic, refer to [Node.js agent](/docs/serverless-function-monitoring/azure-function-monitoring/container/#nodejs).
    * To instrument your `.NET` runtime with the New Relic, refer to [.NET agent](/docs/serverless-function-monitoring/azure-function-monitoring/container/#net).
    * To instrument your `Python` runtime with the New Relic, refer to [Python agent](/docs/serverless-function-monitoring/azure-function-monitoring/container/#python).

    </TabsPageItem>
    
  </TabsPages>
</Tabs>


</Step>

<Step>
## Configure environment variables [#env-variables]

After you've instrumented your Azure Function with the New Relic .NET Agent, [configure environment variables.](/docs/serverless-function-monitoring/azure-function-monitoring/env-variables-azure)

</Step>

<Step>

## Restart your Azure Function [#restart]

After you've added the environment variables, restart your Azure Function to apply the changes.
</Step>

</Steps>
