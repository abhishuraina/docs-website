Our Streaming Video & Ads <DNT>**Summary**</DNT> page helps you troubleshoot issues with the real user performance of your app. Use the Summary page to:

* Get core web vitals at a glance, including **Video Completion Rate** and **Concurrent Views**.
* View trends in an app's performance.
* Quickly identify the key performance indicators that display the **Video quality** and the **Video engaegment metrics**.
* Go directly to other browser UI pages to examine problems and issues in more detail.
* Add recommended alert conditions.

## View the Summary page

To view the summary of the media agent for a specific application:

1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com) > All Entities**</DNT>.
2. Click <DNT>**Video Player**</DNT>> 
2. Select the specific application.

## Dimensional KPIs

Explore and analyze traffic in depth by utilizing multiple parameters, including platform and video player name. Gain insights into customer viewing habits, such as the percentage of customers watching specific content, the resolution at which videos are being viewed, and detailed information about the device's operating system version.

* **Platform**: The device type for example, Roku, Browser, Android, or iOS device.

* **OS Version**: The operating system version for the device. If you are using an Android or iOS device, the version of the operating system will be displayed.

* **Video player name**: The name of the video player used to play the video.

* **ISP**: The Internet Service Provider (ISP) used by the viewer.

* **Region**: The geographical location of the viewer.

* **Video Resolution**: The resolution of the video played by the viewer.

* View the sessions of the customer and troubleshoot users issue by clicking the **Analyse user session**, which will provide you with the detailed information about the user sessions, such as:

    * **Session ID**: The unique identifier for the session.
    * **User ID**: The unique identifier for the user.
    * **Application Name**: The name of the application that was entered at the time of installation.
    * **Content Title**: The title of the content that was viewed.
    * **Session Start**: The time at which the session started.
    * **Session End**: The time at which the session ended.
    * **Platform**: The type of device used by the user.
    * **Errors**: The errors that occurred during the session.

You will be able to drill down into specific KPIs for more detailed views.

For example, when you click "Video Start Failures" should show a breakdown by device type or geographical location.
