### New doc

* Added support for [AWS SQS message queue instrumentation](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) in the Java Agent, including distributed trace headers for SQS messages starting from version 2.1.0, with guidance on custom instrumentation for message receiver operations.
* [Alert Intelligence Tool](/docs/agentic-ai/tools/alert-intelligence-tool) enhances the ability to assess application health and performance through the agentic integration with our partners.

### Major changes

* Added [Amazon CloudWatch Metric Streams](docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams), [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached), and [Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) are [Supported entity relationships](docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) for OpenTelemetry.

### Minor change

* The Distributed Tracing for sending messages is supported from 2.1.0 to latest in the [framework and libraries](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#frameworks) for Java monitoring.
* AWS Glue Spark ETL jobs limitations for Python Shell jobs for [AWS Glue monitoring integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-glue-monitoring-integration).

### Release note

Stay up-to-date on our most recent release:

* [.NET agent v10.36.0](/docs/release-notes/agent-release-notes/net-release-notes/net-agent-10-36-0/)
    * Added .NET 8+ support for System.Data.Odbc.
    * Added support for instrumenting OpenSearchClient requests.
    * Fixed Azure Function app name prefix to `faas.name` attribute in Azure Function transactions.
    * Fixed the custom attribute values of type float and decimal were not serialized correctly.
    * Improved the error handling in AWS account ID parsing logic.

* [Browser agent v1.281.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.281.0/)
    * Enhanced the `UserAction` fields to capture "nearest" field values when the `UserAction` target does not include the desired fields.

* [Browser agent v1.280.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.280.0/)
    * Remove agentIdentifier argument from agent constructors.
    * Roll back to previous FirstInteraction implementation.

* [Node.js agent v12.12.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-12-12-0/)
    * Added health check support for upcoming features.
    * Added OpenTelemetry bridge instrumentation, which includes a context manager and processor to handle synthesizing segments and time-slice metrics.
    * Updated default config to accept undefined as default value.
    * Updated the compatibility report.
