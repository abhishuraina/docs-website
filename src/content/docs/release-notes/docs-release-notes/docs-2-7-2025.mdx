### New doc

* [AWS SQS message queue instrumentation](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) support added in the Java Agent, including distributed trace headers for SQS messages starting from version 2.1.0, with guidance on custom instrumentation for message receiver operations.
* [Alert Intelligence Tool](/docs/agentic-ai/tools/alert-intelligence-tool) enhance the ability to assess application health and performance through the agentic integration with our partners.
* [Confluent Cloud managed streaming for Apache Kafka](/docs/infrastructure/other-infrastructure-integrations/confluent-cloud-integration) provide guidance on monitoring Kafka clusters and topics, and how to set up alerts for Confluent Cloud.
* [New Relic AI summary for Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/nrai-summary) helps summarize and map session replay event trail data, enabling a better understanding of errors and successes in your user's browser journey.
* [Response intelligence with New Relic AI](/docs/alerts/incident-management/response-intelligence-ai) reduces the time to resolve issues and incidents by using New Relic AI with response intelligence.
* [Browser logs](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview) feature captures real-time browser console logs to enhance observability, simplify debugging, and improve user experience.
* [Introduction to Transaction 360](/docs/apm/transactions/workload-performance-monitoring/introdution) offers a revolutionary approach to application monitoring and troubleshooting by automatically capturing and correlating all entities and data related to a specific problem or question.
* [AWS logs in Context](/docs/logs/logs-context/aws-logs-in-context) offers users a comprehensive, integrated log management solution that enhances both troubleshooting efficiency and system observability within AWS environments.
* Overview of the Agent Control [authentication](/docs/new-relic-control/agent-control/authentication) process.
* Manage and reinstall your current instrumentation with [Agent Control](/docs/new-relic-control/agent-control/reinstrumentation)
* [Agentic Control](/docs/new-relic-control/agent-control/overview) is a lightweight agent supervisor that manages other New Relic and New Relic distribution of OpenTelemetry agents by communicating with Fleet Control to remotely configure, update, and monitor the health of infrastructure-level instrumentation.
* [Pipeline control](/docs/new-relic-control/pipeline-control/overview) allows you to manage the flow of data from your monitored entities to New Relic.
* [Docker container](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration) integration monitors and reports metrics from docker containers to New Relic.

### Major changes

* Added [Amazon CloudWatch Metric Streams](docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams), [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached), and [Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) are [Supported entity relationships](docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) for OpenTelemetry.

### Minor change

* The Distributed Tracing for sending messages is supported from 2.1.0 to latest in the [framework and libraries](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#frameworks) for Java monitoring.
* AWS Glue Spark ETL jobs limitations for Python Shell jobs for [AWS Glue monitoring integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-glue-monitoring-integration).
* Added Canada (British Columbia and Saskatchewan), Chicago, District of Columbia, illinois, and Vermont to the territories that are currently subject to [sales tax](/docs/accounts/accounts/billing/view-or-change-account-tax-information)
* Updated the Windows option for [instrumenting your Azure Functions with the New Relic .NET Agent](/docs/serverless-function-monitoring/azure-function-monitoring/install-serverless-azure-monitoring/#instrument-your-azure-functions-with-the-new-relic-net-agent).
* New Relic Java's layered and SDK instrumentation are now compatible with `com.amazonaws:aws-lambda-java-events` versions 3.0.0 to the latest.
* Added exceptional cases where we might see both [legacy externals and uninstrumented guids](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/#handling-legacy-externals-and-uninstrumented-entities-exceptions-in-entity-relationships)
* Added instruction how the user can onboard to the [new map](/docs/service-architecture-intelligence/maps/advanced-maps) experience directly from the Service Maps page.
* Updated the [supported hour](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings) section.
* [dSYM](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps) files are now limited to 1 GB each.
* Added `NODE_OPTIONS` as an additional [environment variable for AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda/#nodejs).
* Added [log labels](/docs/apm/agents/java-agent/configuration/java-agent-config-file-template) to agent config and [agent forwarded log label](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file) information.
* Updated the [New Relic api keys](/docs/apis/intro-apis/new-relic-api-keys) about how to manage and obtain full keys when needed.
* Users of [Java Agent 8.18.0](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) need to manually enable SQS instrumentation. You can enable this instrumentation in your `newrelic.yml` file.
* The `NEW_RELIC_USE_ESM` will be removed from the future versions of the Node.js [lambda layers](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda)
* Updated the [conditions per policy](/docs/alerts/admin/rules-limits-alerts) limit from 500 to 1000.
* Effective Monday, March 3, 2025, our [Kubernetes integration](/eol/2025/03/deprecation-notice-v1.27-and-lower) will not support Kubernetes v1.27 and earlier. The Kubernetes integration v3.40.1 and later will be compatible only with Kubernetes versions 1.28 and later.
* Restored the [Mermaid digram support](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#markdown) information.
* Added the associated pricing details for [Advanced and Core Compute pricing plans](/docs/alerts/create-alert/create-alert-condition/alert-conditions)
