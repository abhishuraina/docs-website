### Major changes

* New Relic has discontinued the use of the following IP ranges:
  * 3.145.244.128/25
  * 3.77.79.0/25
  * 3.27.118.128/25
  * 20.51.136.0/25
  * 4.197.217.128/25
  * 18.246.82.0/25
  * 158.177.65.64/29
  * 159.122.103.184/29
  * 161.156.125.32/28
  
  As of May 1, 2025, these ranges may be reallocated by the cloud provider to other customers for purposes beyond our control. Please update your [network](/docs/new-relic-solutions/get-started/networks) configurations accordingly.

* Added a troubleshooting document to [retrieve the New Relic license key](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/retrieve-license) for issues using AWS Secrets Manager with New Relic.
* Added a troubleshooting document for [Lambda functions not showing as instrumented](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-not-instrumented-lambda-function).
* Added a troubleshooting document for issues [adding multiple AWS accounts with respective regions](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-aws-lambda).
* Added a troubleshooting document for issues [using AWS Secrets Manager with New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-unable-to-add-multiple-aws-accounts).

### Minor changes

* [dSYM](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps) files are now limited to 1 GB each.
* Added `NODE_OPTIONS` as an additional [environment variable for AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda/#nodejs).
