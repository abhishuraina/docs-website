          <TabsPageItem id="1">
            1. Enter a name for your application, and click **Save and continue** to create the application.

            2. On the "Configure the browser agent" page, click **Save and continue**.

            3. The browser JavaScript snippet is generated in the UI. Once you have the snippet, paste it as close to the top of your `<head>` tag as possible, but after any position-sensitive META tags (such as `X-UA-Compatible` and `charset`). Click **Continue**.

            For detailed information about placing the snippet, see "JavaScript placement requirements"
            
            <CollapserGroup>
              <Collapser
                id="JavaScript placement requirements"
                title="JavaScript placement requirements"
              >
                Injecting the JavaScript inline in the HEAD is an unusual requirement for a JavaScript library, and different from how third-party scripts are typically included. We require this so that browser provides accurate information without impacting page load performance.

                The injected browser JavaScript elements wrap many of the browser's built in APIs to record information about JavaScript errors or callback timings. The unusual placement of the code element is necessary for the following reasons:

                * The inline HEAD placement ensures the instrumentation code is loaded before all other scripts so that wrapping will occur when other libraries are registered.
                * Inline code placement also eliminates the network round trips caused by externally referenced scripts.
              </Collapser>
            </CollapserGroup>

            4. Click **Continue** when asked where you want to install the media agent.

            5. Add the video agent to your application by copying the code snippet and pasting it as close to the top of your `<head>` tag as possible, but after any position-sensitive META tags (such as `X-UA-Compatible` and `charset`).

              <Callout variant="important">
              Include the downloaded script in your application, ensuring it is placed below the existing browser agent code.
              </Callout>

            With an APM agent, you can deploy media streaming agent without manually adding the snippet to your code. When you restart the agent, it automatically uses the latest version of the media agent.

            There are specific steps for each APM agent:

            * [Java](/docs/agents/java-agent/instrumentation/page-load-timing-java)
            * [.NET](/docs/agents/net-agent/features/page-load-timing-net)
            * [PHP](/docs/agents/php-agent/features/page-load-timing-php)
            * [Ruby](/docs/agents/ruby-agent/features/page-load-timing-ruby)
            * [Python](/docs/agents/python-agent/supported-features/page-load-timing-python)
            * [Node.js](/docs/agents/nodejs-agent/supported-features/page-load-timing-nodejs)
          </TabsPageItem>

          <TabsPageItem id="3">
            If you'd prefer to create your browser snippet programmatically, you can do so via our NerdGraph, our GraphQL API.

            For more on this, see our [NerdGraph tutorial](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/) to create a browser snippet and configure it.

            Once you have the snippet, add it to the `<head>` tag of your app.
