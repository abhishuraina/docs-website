            <Callout variant="important">
              Choose this method if you're not monitoring your application with New Relic yet.
            </Callout>
          
            1. Enter a name for your application, this creates an entity in New Relic that represents your application.
            2. Click **Save and continue**.
            3. On the **Configure the browser agent** page, by default the **Turn on distributed tracing** is enabled, you may toggle Capture sessions replay on to track users interactions with the browser app.
            4. Click **Save and continue**.
            5. The browser JavaScript snippet is generated in the UI. Paste it as close to the top of your `<head>` tag as possible, but after any position-sensitive META tags (such as `X-UA-Compatible` and `charset`). Click **Continue**.

            For detailed information about placing the snippet, see [JavaScript placement requirements](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-browser-monitoring/#javascript-placement)

            6. You will be redirected to the **Installation media streaming agent** page. Click **Continue**.
            7. Add the video agent to your application by copying the code snippet and pasting it below the existing browser agent code.

          <TabsPageItem id="2">

            1. Set the APM congifuration to automatically inject the media agent into your application.

            2. With an APM agent, you can deploy media streaming agent without manually adding the snippet to your code. When you restart the agent, it automatically uses the latest version of the media agent.

                There are specific steps for each APM agent:

                * [Java](/docs.newrelic.com/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/#bm-auto_instrument)
                * [.NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration/#browsermon-autoInstrument)
                * [PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-autorum)
                * [Ruby](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/#browser_monitoring-auto_instrument)
                * [Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration/#browser-auto)
                * [Node.js](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/browser-monitoring-nodejs-agent)

            3. Choose the application you want to see th Browser data from the drop-down.
            4. Click **Continue**.
            5. Click **Save and continue** on the **Configure the browser agent** page.
            6. Click **Continue** to start gathering the data.
            7. Click **Continue** when asked where you want to install the media agent.
            8. Add the video agent to your application by copying the code snippet and pasting it as close to the top of your `<head>` tag as possible, but after any position-sensitive META tags (such as `X-UA-Compatible` and `charset`).

              ```json
              curl -L -o newrelic-video-bitmovin.latest.min.js https://download.newrelic.com/media-agents/browser/bitmovin/newrelic-video-bitmovin.latest.min.js 
              ```
            9. Include the downloaded script in your application, ensuring it is placed below the existing browser agent code.
            10. Add the following code snippet to your application to start monitoring your video player.

              ```json
                // var player = bitmovin('player');

                nrvideo.Core.addTracker(new nrvideo.BitmovinTracker(player));
              ```
            11. Click <DNT>**See your data**</DNT> to deploy your updated code and see the data.
          </TabsPageItem>

          <TabsPageItem id="3">
            If you'd prefer to create your browser snippet programmatically, you can do so via our NerdGraph, our GraphQL API.

            For more on this, see our [NerdGraph tutorial](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/) to create a browser snippet and configure it.

            Once you have the snippet, add it to the `<head>` tag of your app.
          </TabsPageItem>
